چند نکته

هرچند مطالبی که می‌گذارم سعی می‌کنم فارسی باشه، ولی قسمت عمده‌ای از برنامه نویسی با زبان انگلیسیه، و زبان پایتون هم خیلی به انگلیسی نزدیکه. و به هر حال اگر بخواید منابع و داکیومنت‌های پایتون رو نگاه کنید و یا تو سایت‌هایی مثل stack overflow بگردید و سؤال بپرسید باید انگلیسی بلد باشید.

مورد دیگه تمرین زیاده. یادگرفتن یه زبان برنامه‌نویسی مثل زبان‌های دیگه فراره و اگه مداوم نباشه سریع فراموش میشه. مثلا اونایی که درس می‌خونن از چیزایی که اینجا یاد می‌گیرن می‌تونن توی درسشون استفاده کنن. مثلا می‌تونید برنامه‌ای بنویسید که فرمول‌های ریاضی یا فیزیک رو براتون حل بکنه. یا می‌تونید واسه شروع بازی‌های متنی ساده‌ای درست کنید، شبیه‌سازی انجام بدید، یا با استفاده از پکیج‌هایی مثل PySimpleGUI که خیلی ساده هستن محیط گرافیکی یک نرم‌افزار رو درست کنید، یا هر چیز دیگه‌ای که بهش علاقه دارید!

منابع مهم Python

مهم‌ترین منابع پایتون به نظرم این‌ها هستن (به ترتیب اولویت):
  1. خود IDE: توی PyCharm اگه ماوس رو روی تابع یا تقریبا هرچیزی نگه دارید، داکیومنت مربوط به اون نشون داده میشه. اگر ctrl رو نگه دارید و روش کلیک کنید معمولا به منبع اون تابع یا ... هدایت میشه که می‌تونید ببینید چطور ساخته شده و چطور باید ازش استفاده کرد. تو شروع شاید یه کم گیج کننده باشه، ولی فوق‌العاده کاربردیه.
  2. docs.python.org : توی سایت Python می‌تونید با جزئیات کامل ببینید که هر متد، تابع، عملگر و ... چه کاری انجام میدن، متدهای مختلف هر کلاس رو ببینید و ... این سایت خیلی کامله. اگر دنبال چیز خاصی می‌گردید حتما قبل از اینکه تو گوگل سرچ کنید اینجا رو نگاه کنید. (آموزش پایتون هم قرار دادن که اگه حوصله خوندنش رو داشته باشید خوبه)
  3. stackoverflow : اگر توی کدتون به مشکلی برخوردید یا سؤالی دارید، احتمال خیلی زیاد یکی قبل از شما این مشکل رو داشته و اینجا پرسیده. اینجا می تونید راه حل خیلی از مسائل مربوط به برنامه‌نویسی رو پیدا کنید. ولی توصیه می‌کنم مستقیم به این سایت نرید و یا مشکلتون رو سرچ نکنید. سعی کنید به روش‌های دیگه (روش‌های بالا و یا حتی سعی و خطا) راه حل رو پیدا کنید تا مهارت برنامه‌نویسی‌تون رو تقویت کنید.
  4. سایت‌های دیگه‌ای مثل w3school و realpython.com هم سایتای خوبی هستن! (هرچند سایت خودمون قراره بهتر بشه 😁)
