چند نکته

هرچند مطالبی که می‌گذارم سعی می‌کنم فارسی باشه، ولی قسمت عمده‌ای از برنامه نویسی با زبان انگلیسیه، و زبون پایتون هم خیلی به انگلیسی نزدیکه. اگر بخواید منابع و داکیومنت‌های پایتون رو نگاه کنید و یا تو سایت‌هایی مثل stack overflow بگردید و سؤال بپرسید باید انگلیسی بلد باشید. پس اگر واقعا می‌خوایم برنامه‌نویس باشیم باید از همین الآن روی بهبود زبان انگلیسیمون کار کنیم.

مورد دیگه تمرین زیاده. یادگرفتن یه زبون برنامه‌نویسی مثل زبون‌های دیگه فراره و اگه مداوم نباشه سریع فراموش میشه. بعضی‌هامون شاغلیم و یعضی‌هامون درس می‌خونیم. اونایی که درس می‌خونن از چیزایی که اینجا یاد می‌گیرن توی درسشون استفاده کنن. مثلا می‌تونید برنامه‌ای بنویسید که فرمول‌های ریاضی یا فیزیک رو براتون حل بکنه. یا می‌تونیم واسه شروع بازی‌های متنی ساده‌ای درست کنیم و ازش لذت ببریم.

منابع مهم Python

مهم‌ترین منابع پایتون به نظرم این‌ها هستن (به ترتیب اولویت):
  1. خود IDE: توی PyCharm اگه ماوس رو روی تابع یا تقریبا هرچیزی نگه دارید، داکیومنت مربوط به اون نشون داده میشه. اگر ctrl رو نگه دارید و روش کلیک کنید معمولا به منبع اون تابع یا ... هدایت میشه که می‌تونید ببینید چطور ساخته شده و چطور باید ازش استفاده کرد. تو شروع شاید یه کم گیج کننده باشه، ولی این واقعا بهترین منبعه به نظرم.
  2. docs.python.org : توی سایت Python می‌تونید با جزئیات کامل ببینید که هر متود، تابع، عملگر و ... جه کاری انجام میدن، متودهای مختلف هر کلاس رو ببینید و ... این سایت واقعا خیلی کامله. اگر دنبال چیز خاصی می‌گردید حتما قبل از اینکه تو گوگل سرچ کنید انجا رو نگاه کنید. آموزش پایتون هم قرار دادن که واسه شروع خیلی خوبه.
  3. stackoverflow : اگر توی کدتون به مشکلی برخوردید. سؤال دارید احتمال خیلی زیاد یکی قبلا از شما این مشکل رو داشته و اینجا پرسیده. اینجا می تونید راه حل خیلی از مسائل مربوط به برنامه‌نویسی رو پیدا کنید. ولی توصیه می‌کنم مستقیم به این سایت نرید و یا مشکلتون رو سرچ نکنید. سعی کنید به روش‌های دیگه (روش‌های بالا و سعی و خطا) راه حل رو پیدا کنید تا مهارت برنامه‌نویسی‌تون رو تقویت کنید.
